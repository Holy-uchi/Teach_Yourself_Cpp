# 第1章 イテレータ・範囲・参照の作法（8問）

## 第1問 half-open range の感覚を身につける

【問題】
以下の仕様を満たすコードを書け。

【与えられるもの】

- `std::vector<int> v = {10, 20, 30, 40, 50, 60, 70};`

【要件】

- `v` の **先頭から3番目の要素以上、6番目の要素未満** の範囲だけを走査し、その要素を順に標準出力せよ
- 期待される対象要素は `30, 40, 50` である
- `begin()` と `end()` の差ではなく、**iterator で表現された部分範囲**として書くこと

【制約】

- 添字アクセスは禁止
- `for` 文は使ってよいが、走査範囲は iterator で指定すること
- `std::next` を使わず、まずは `begin()` からの iterator 操作だけで書くこと

【この問題の主眼】

- `[first, last)` という half-open range の感覚に慣れること
- `last` 自体は含まれない、というSTLの基本作法を体で理解すること

【この問題の主眼】

- 「読むだけなら const な範囲を使う」という作法に慣れること
- iterator の const 性が、操作可能性をどう制限するかを理解すること

【この問題の主眼】

- range-based for で `auto x : container` と書いたときのコピー罠を理解すること
- 要素更新が目的なら `auto&` が基本になることを体得すること

## 第8問 `end()` は要素ではないことを明確に理解する

【問題】
以下の仕様を満たすコードを書け。

【与えられるもの】

- `std::vector<int> v = {11, 22, 33, 44};`

【要件】

- `v.begin()` が先頭要素を指すことを確認せよ
- `std::prev(v.end())` が末尾要素を指すことを確認せよ
- `v.end()` 自体は要素を指していないことを、コード上の扱いで明確にせよ
- 「末尾要素を取りたいときに `end()` をそのまま参照しない」という正しい書き方で、末尾要素を標準出力せよ

【制約】

- `back()` は使わないこと
- `std::prev` を使うこと
- `end()` を直接参照する危険なコードは完成版に含めないこと

【この問題の主眼】

- `end()` は「最後の要素」ではなく「最後の次」を表す iterator であることを定着させること
- STLの境界感覚を曖昧にしないこと

## 第50問 `min_element` / `max_element` / `equal` などの基本実用algorithm

【問題】

【与えられるもの】

- `std::vector<int> a = {8, 3, 15, 6, 10};`
